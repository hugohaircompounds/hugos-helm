# Release 0.2.0

Planning document for the next minor release. Two user-facing features bundled together:

1. **Task customization** — proper status dropdown, reorderable groups, collapsible groups, filters
2. **Timesheet redesign** — detail pane in the middle column, day-at-a-glance timeline bar

Built tomorrow, shipped as `0.2.0` via `npm version minor` → `npm run release`.

---

## Why now

The task section is currently functional but inflexible — status is free-text (ClickUp rejects invalid statuses silently), groups are unsortable, and there's no filtering. The timesheet has inline-edit clutter that doesn't match the task UX, and no visual sense of how the day is laid out. Both have been growing friction points.

**Outcome**: Tasks become a real triage surface. Timesheet gets a timeline visualization and a cleaner detail-pane edit flow matching the task pane.

**Explicitly out of scope for this release**: scheduler, timer logic, theme system, auth flow, Google/Gmail panels.

---

## Feature 1 — Task customization

### 1a. Status dropdown (replace free-text)

**Problem**: `src/components/TaskDetail.tsx:170-177` is a free-text `<input>`. ClickUp rejects invalid statuses silently.

**Blocker**: The ClickUp service doesn't currently expose valid statuses for a list. Must add this first.

**New main-process work**:
- `electron/services/clickup.ts` → add `getListStatuses(listId: string): Promise<ListStatus[]>`. Calls `GET /list/{listId}` and extracts the `statuses` array (ClickUp already includes it in the list object).
- `electron/ipc/handlers.ts` → register `ipcMain.handle('list:statuses', ...)`.
- `electron/preload.ts` → expose `getListStatuses(listId)`.
- `shared/types.ts` → export `ListStatus` interface, extend `HelmApi` with `getListStatuses: (listId: string) => Promise<ListStatus[]>`.

**Renderer change**:
- `TaskDetail.tsx` → replace free-text input with `<select>`. Fetch statuses once per `detail.listId` via a new `useListStatuses(listId)` hook that caches results. Show a colored dot next to each option using ClickUp's per-status color (reuse the `--status-color` CSS-variable pattern we already introduced).

### 1b. Reorderable status groups

**Current state**: `TaskList.tsx:39-48` groups tasks via a `Map<string, {color, tasks}>`, order = task-insertion order. Not persisted.

**Changes**:
- Render **▲ ▼** buttons in each group header. Clicking moves the group up/down by one.
- Persist the order in a new setting `taskStatusGroupOrder: Record<string, string[]>` (keyed by list id or `'__all__'` for the cross-list view).
- Groups not in the stored order fall back to ClickUp's `orderindex`.

### 1c. Collapse/expand status groups

**Changes**:
- Add a disclosure triangle to each group header.
- Collapsed state persisted in a new setting `collapsedStatusGroups: string[]`.
- Collapsed groups render only the header (count still visible).
- Debounced save — reuse the pattern in `src/hooks/useLayout.ts`.

### 1d. Filters

**Changes**:
- New component `src/components/TaskFilterBar.tsx` — collapsible panel at the top of the task list. Contents:
  - Multi-select chips for statuses
  - Multi-select chips for list names (work categories)
  - Multi-select chips for priorities (Urgent/High/Normal/Low)
  - Date range pair (from / to) for due date
- Filter state persisted in new setting `taskFilters: { statuses; listNames; priorities; dueFrom; dueTo }`.
- `TaskList.tsx` applies filters in the `useMemo` before grouping.
- Active-filter count badge on the filter toggle when the panel is collapsed.

## Feature 2 — Timesheet redesign

### 2a. Detail pane in middle column

**Current state**: `TimesheetEditor.tsx:146-183` has an inline expandable edit form per row. Middle column shows placeholder text at `App.tsx:98` when the Timesheet tab is active.

**Changes**:
- New component `src/components/TimeEntryDetail.tsx` — mirrors `TaskDetail.tsx` structure: `panel-header`, pills (duration, date), three edit fields (description, start, end). Same `save()` pattern as the current inline form (onBlur → IPC, defensive merge).
- `App.tsx`:
  - Add `const [selectedEntryId, setSelectedEntryId] = useState<string | null>(null);`
  - Middle column: when `tab === 'timesheet'` and `selectedEntryId` is set, render `<TimeEntryDetail>` instead of placeholder.
  - Lift the entries list state into `App.tsx` (or new `src/hooks/useTimeEntries.ts`) so both `TimesheetEditor` and `TimeEntryDetail` share it. Matches the existing `useClickUp` pattern.
- `TimesheetEditor.tsx`:
  - Remove inline edit form (lines 146-183) and the `editingId` state.
  - Rows become clickable → `setSelectedEntryId(e.id)`. Add selected-highlight styling matching `TaskList.tsx`.
  - Keep the Delete button.

### 2b. Timeline bar visualization

**New component**: `src/components/TimelineBar.tsx`, rendered at the top of `TimesheetEditor`, above the entry list.

**Geometry**:
- Base range: **8:00 AM – 5:00 PM today** (9 hours).
- **Auto-extend**: if any entry's `start < 8:00` or `end > 17:00`, widen the range to cover it. 8–17 is the minimum.
- **Lunch band**: ghost layer 13:00–14:00 with a subtle `rgb(var(--ink-muted) / 0.1)` background, hairline "Lunch" label. Real entries render on top.
- Each `TimeEntry` renders as an absolutely-positioned `<div>` on the bar.
- Hour ticks with labels (8am, 9am, …) at top of bar.

**Color assignment — hash-derived HSL**:
- New helper `src/utils/entryColor.ts`:
  ```
  hash(taskId) → hue [0, 360)
  HSL → hsl(hue, var(--entry-sat), var(--entry-light))
  ```
- Per-theme `--entry-sat` and `--entry-light` defined in `index.css`. Dark default: 72% / 58%. Light default: 65% / 45%. Bolder themes (Cyberpunk, Neon) get tuned values.
- Entries with no `taskId` (manual blocks) use `--ink-muted` at 40% opacity.
- Same `taskId` always gets the same color within a theme.

**Labels on segments**:
- Task title inside the segment if width > ~80px; otherwise show on hover via `title` attribute.
- Start time at the segment's left edge, end time at right edge (small `text-xs` labels).

**Overlap detection & striped rendering**:
- Sort entries by `start`. Detect overlaps with `entries[i].end > entries[j].start && entries[i].start < entries[j].end`.
- Render overlapping spans as a single striped segment:
  ```
  background: repeating-linear-gradient(45deg, colorA 0 8px, colorB 8px 16px);
  ```
- Label as `{taskA} / {taskB} (overlapped)`.

**Click behavior**: clicking a segment sets `selectedEntryId` (same interaction model as the list).

**Edge case — running timer**: pulsing animation on the in-progress entry's right edge; visible end = `Date.now()`, re-computed every 30s.

## New settings

Extend `Settings` interface in `shared/types.ts`, `DEFAULT_SETTINGS` in `electron/db/index.ts`, and the `stored[...]` reads + writes in `getSettings`/`saveSettings`:

```typescript
taskStatusGroupOrder: Record<string, string[]>;    // listId/'__all__' → ordered status names
collapsedStatusGroups: string[];                    // collapsed status names
taskFilters: {
  statuses: string[];
  listNames: string[];
  priorities: number[];    // 1|2|3|4
  dueFrom: number | null;  // unix ms
  dueTo: number | null;
};
```

Defaults: empty objects/arrays/nulls.

## Release bundling

Single minor version bump (user-visible new functionality, no breaking changes):

```powershell
git add -A ; git commit -m "feat: task customization + timesheet redesign"
npm version minor
npm run release
git push --follow-tags
```

Installed Helm picks up `0.2.0` on next launch. See [WORKFLOW.md](./WORKFLOW.md) for the full workflow reference.
